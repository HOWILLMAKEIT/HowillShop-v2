# HowillSHOP 部署指南

> 本文档为 HowillSHOP（版本二）的完整部署指南，适用于 CentOS 8 / Ubuntu 20.04+ 服务器环境。

---

## 1. 服务器环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| OS | CentOS 8 / Ubuntu 20.04+ | 推荐阿里云 ECS |
| JDK | OpenJDK 11 | `java -version` 验证 |
| MySQL | 8.0+ | UTF8MB4 字符集 |
| Tomcat | 9.x | WAR 部署 |
| Nginx | 1.18+ | 反向代理（80 → 8080） |
| Maven | 3.9+ | 本地构建用 |

---

## 2. 本地构建

在本地项目目录下执行：

```bash
mvn -DskipTests package
```

构建成功后，WAR 包位于 `target/howill-shop-2.0.1.war`。

---

## 3. 服务器基础环境安装

### 3.1 安装 JDK 11

```bash
# CentOS
sudo yum install -y java-11-openjdk java-11-openjdk-devel

# Ubuntu
sudo apt install -y openjdk-11-jdk

# 验证
java -version
```

### 3.2 安装 MySQL 8

```bash
# CentOS
sudo yum install -y mysql-server
sudo systemctl start mysqld
sudo systemctl enable mysqld

# Ubuntu
sudo apt install -y mysql-server
sudo systemctl start mysql
sudo systemctl enable mysql
```

设置 root 密码：

```bash
mysql -u root -p
ALTER USER 'root'@'localhost' IDENTIFIED BY '你的数据库密码';
FLUSH PRIVILEGES;
```

### 3.3 安装 Tomcat 9

```bash
# 创建 tomcat 专用用户
sudo useradd -r -m -d /opt/tomcat -s /bin/false tomcat

# 下载并解压
cd /tmp
wget https://dlcdn.apache.org/tomcat/tomcat-9/v9.0.105/bin/apache-tomcat-9.0.105.tar.gz
sudo mkdir -p /opt/tomcat
sudo tar xzf apache-tomcat-9.0.105.tar.gz -C /opt/tomcat --strip-components=1

# 设置权限
sudo chown -R tomcat:tomcat /opt/tomcat
sudo chmod -R 755 /opt/tomcat/logs
```

### 3.4 安装 Nginx

```bash
# CentOS
sudo yum install -y nginx

# Ubuntu
sudo apt install -y nginx

sudo systemctl start nginx
sudo systemctl enable nginx
```

## 4. 数据库初始化

### 4.1 上传 SQL 文件

将项目中的 `db/` 目录上传到服务器：

```bash
scp -r db/ root@你的服务器IP:/home/HowillSHOP/db/
```

### 4.2 按顺序执行 SQL 脚本

```bash
# Step 1: 建库 + 基础表（版本一原有）
mysql -u root -p < /home/HowillSHOP/db/schema.sql

# Step 2: 插入种子数据（基础商品和订单数据）
mysql -u root -p javaweb_shop < /home/HowillSHOP/db/seed.sql

# Step 3: 版本二迁移（新增 4 张日志表 + Admin 角色 + 索引）
mysql -u root -p javaweb_shop < /home/HowillSHOP/db/migration_v2.sql

# Step 4: 插入版本二测试数据（商家/用户账号 + 日志数据）
mysql -u root -p javaweb_shop < /home/HowillSHOP/db/test_data_fixed.sql
```

> **重要**：必须按顺序执行。`migration_v2.sql` 创建日志表和 Admin 账号，`test_data_fixed.sql` 插入完整测试数据（商家、用户、订单及各类日志），让数据分析和推荐系统有数据可展示。

### 4.3 验证数据库

```bash
mysql -u root -p javaweb_shop -e "SHOW TABLES;"
# 应看到：users, products, categories, orders, order_items,
#         user_login_logs, user_browse_logs, user_purchase_logs, operation_logs

mysql -u root -p javaweb_shop -e "SELECT id, username, role FROM users WHERE role='ADMIN';"
# 应输出：1 | admin | ADMIN
```

## 5. 配置环境变量

### 5.1 创建 .env 文件

```bash
sudo mkdir -p /home/HowillSHOP
sudo vim /home/HowillSHOP/.env
```

写入以下内容（根据实际情况修改）：

```bash
# === 数据库 ===
DB_DRIVER=com.mysql.cj.jdbc.Driver
DB_URL=jdbc:mysql://localhost:3306/javaweb_shop?useSSL=false&serverTimezone=UTC&characterEncoding=utf8&allowPublicKeyRetrieval=true
DB_USERNAME=root
DB_PASSWORD=你的数据库密码
DB_POOL_MAX_SIZE=20
DB_POOL_MIN_IDLE=5
DB_POOL_CONNECTION_TIMEOUT_MS=30000
DB_POOL_IDLE_TIMEOUT_MS=600000
DB_POOL_MAX_LIFETIME_MS=1800000

# === 邮件服务（QQ 邮箱 SMTP）===
MAIL_SMTP_HOST=smtp.qq.com
MAIL_SMTP_PORT=587
MAIL_SMTP_AUTH=true
MAIL_SMTP_STARTTLS=true
MAIL_USERNAME=你的QQ邮箱@qq.com
MAIL_PASSWORD=你的QQ邮箱授权码
MAIL_FROM=你的QQ邮箱@qq.com

# === 阿里云 OSS（商品图片存储）===
OSS_ENDPOINT=https://oss-cn-beijing.aliyuncs.com
OSS_BUCKET=你的bucket名称
OSS_ACCESS_KEY_ID=你的AccessKeyId
OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
OSS_BASE_URL=https://你的bucket.oss-cn-beijing.aliyuncs.com/
OSS_PREFIX=products/
OSS_SIGN_EXPIRE_SECONDS=3600
```

### 5.2 设置文件权限

```bash
sudo chown tomcat:tomcat /home/HowillSHOP/.env
sudo chmod 600 /home/HowillSHOP/.env
```

## 6. 部署 WAR 包

```bash
# 1. 上传 WAR 到服务器
scp target/howill-shop-2.0.1.war root@你的服务器IP:/tmp/

# 2. 清除旧版本（如有）
sudo rm -rf /opt/tomcat/webapps/shop
sudo rm -f /opt/tomcat/webapps/shop.war

# 3. 部署新版本
sudo cp /tmp/howill-shop-2.0.1.war /opt/tomcat/webapps/shop.war
sudo chown tomcat:tomcat /opt/tomcat/webapps/shop.war
```

> WAR 文件重命名为 `shop.war`，访问路径为 `http://域名/shop/`。

## 7. 配置 Tomcat systemd 服务

### 7.1 查找 JAVA_HOME

```bash
readlink -f $(which java) | sed 's:/bin/java::'
# 例如输出：/usr/lib/jvm/java-11-openjdk-11.0.25.0.9-2.0.1.1.al8.x86_64
```

### 7.2 创建 service 文件

```bash
sudo vim /etc/systemd/system/tomcat.service
```

写入（`JAVA_HOME` 替换为上一步查到的路径）：

```ini
[Unit]
Description=Apache Tomcat 9
After=network.target

[Service]
Type=forking
User=tomcat
Group=tomcat
Environment="JAVA_HOME=你的JAVA_HOME路径"
Environment="CATALINA_HOME=/opt/tomcat"
Environment="CATALINA_BASE=/opt/tomcat"
Environment="CATALINA_PID=/opt/tomcat/temp/tomcat.pid"
Environment="JAVA_OPTS=-Denv.file=/home/HowillSHOP/.env"
ExecStart=/opt/tomcat/bin/startup.sh
ExecStop=/opt/tomcat/bin/shutdown.sh
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

> **关键**：`JAVA_OPTS=-Denv.file=/home/HowillSHOP/.env` 让应用读取 .env 文件覆盖默认的 `db.properties` / `mail.properties` / `oss.properties`。

### 7.3 启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable tomcat
sudo systemctl start tomcat

# 检查状态
sudo systemctl status tomcat
```

## 8. 配置 Nginx 反向代理

### 8.1 创建站点配置

```bash
sudo vim /etc/nginx/conf.d/shop.conf
```

写入（`server_name` 替换为你的域名或 IP）：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    # 反向代理到 Tomcat
    location /shop/ {
        proxy_pass http://127.0.0.1:8080/shop/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # sendBeacon 请求支持
        proxy_http_version 1.1;
        proxy_set_header Connection "";
    }

    # 根路径重定向到 /shop/
    location = / {
        return 302 /shop/;
    }
}
```

### 8.2 测试并生效

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 9. 验证部署

### 9.1 检查服务状态

```bash
sudo systemctl status tomcat
sudo tail -50 /opt/tomcat/logs/catalina.out
ss -tlnp | grep 8080
```

### 9.2 功能验证清单

访问 `http://你的域名或IP/shop/`，逐项检查：

| 功能 | 验证方式 | 预期结果 |
|------|---------|---------|
| 首页商品列表 | 直接访问 | 商品正常显示 |
| 用户注册登录 | 注册新用户 → 登录 | 登录成功，导航栏显示用户名 |
| 管理员登录 | admin / admin123 | 自动跳转管理仪表盘 |
| 管理仪表盘 | /admin/dashboard | KPI 卡片 + 趋势图 + Top10 排行 |
| 数据分析页 | /admin/analytics | 折线图/饼图/异常检测表格 |
| 用户画像 | /admin/profiles | 地域/消费力/偏好分类 |
| 系统日志 | /admin/logs | 4 类日志均有记录 |
| 推荐页面 | 导航栏"推荐" | 显示个性化推荐商品 |
| 商品详情推荐 | 进入任意商品详情 | 底部有"浏览过此商品的人也买了" |
| 浏览数据采集 | 浏览商品详情后查库 | user_browse_logs 有新增记录 |

### 9.3 数据库验证

```bash
mysql -u root -p javaweb_shop -e "
  SELECT '登录日志' AS 表名, COUNT(*) AS 记录数 FROM user_login_logs
  UNION ALL
  SELECT '浏览日志', COUNT(*) FROM user_browse_logs
  UNION ALL
  SELECT '购买日志', COUNT(*) FROM user_purchase_logs
  UNION ALL
  SELECT '操作日志', COUNT(*) FROM operation_logs;
"
```

## 10. 版本一 → 版本二升级说明

如果服务器上已部署版本一，升级步骤：

```bash
# 1. 备份数据库
mysqldump -u root -p javaweb_shop > v1_backup_$(date +%Y%m%d).sql

# 2. 执行版本二迁移（仅新增表，不影响原有数据）
mysql -u root -p javaweb_shop < db/migration_v2.sql

# 3. 插入版本二测试数据
mysql -u root -p javaweb_shop < db/test_data_fixed.sql

# 4. 替换 WAR 包并重启
sudo systemctl stop tomcat
sudo rm -rf /opt/tomcat/webapps/shop /opt/tomcat/webapps/shop.war
sudo cp target/howill-shop-2.0.1.war /opt/tomcat/webapps/shop.war
sudo chown tomcat:tomcat /opt/tomcat/webapps/shop.war
sudo systemctl start tomcat
```

> `migration_v2.sql` 只创建新表和插入 Admin 账号，不会删除或修改版本一的任何现有数据。

```bash

mvn -DskipTests package

# 上传并替换
scp target/howill-shop-2.0.1.war root@你的服务器IP:/tmp/
```

```bash
sudo tail -f /opt/tomcat/logs/catalina.out     # Tomcat 启动日志
sudo tail -f /var/log/nginx/access.log          # Nginx 访问日志
sudo tail -f /var/log/nginx/error.log           # Nginx 错误日志
```

```bash
mysqldump -u root -p javaweb_shop > backup_$(date +%Y%m%d).sql
```
