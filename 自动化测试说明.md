# 自动化功能测试说明

## 概述

本项目使用 AI 辅助生成的自动化测试脚本 `test_checklist.py`，基于 Python requests + BeautifulSoup 模拟真实用户操作，对系统 72 项功能进行黑盒测试和功能测试。

## 测试方法

### 测试类型

- **黑盒测试**：通过 HTTP 请求模拟用户操作，不关注内部实现，只验证输入和输出
- **功能测试**：按功能模块逐项验证，覆盖所有用户角色和业务流程

### 技术实现

| 组件 | 作用 |
|------|------|
| Python requests | 模拟 HTTP 请求（GET/POST），维护 Session 会话 |
| BeautifulSoup | 解析 HTML 响应，验证页面内容和元素 |
| pymysql | 直接查询 MySQL 数据库，验证日志数据完整性 |

脚本通过 `requests.Session` 维持登录状态，模拟真实用户的操作流程（注册→登录→浏览→加购→结算→支付→管理后台），在每一步验证返回页面的内容和 HTTP 状态码。

## 运行方式

### 环境准备

```bash
pip install requests beautifulsoup4 pymysql
```

### 执行测试

确保 Tomcat 已启动，然后运行：

```bash
python test_checklist.py
```

## 测试结果判定

脚本输出两级结果：

- **PASS**（通过）：页面返回正确内容，功能正常
- **FAIL**（失败）：页面缺少预期内容或 HTTP 状态码异常

运行结束后自动汇总通过/失败数量，输出最终结论。
